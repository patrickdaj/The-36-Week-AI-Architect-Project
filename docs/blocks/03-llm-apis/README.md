# 🔌 Unit 3 — LLM APIs

<small>Weeks 5–8 · 2 blocks</small>

[→ Start Block 3](03-first-ai-apps/index.md)

---

You've engineered prompts. You've got your Python sharp. Now you connect the two — calling LLMs programmatically, handling real-world concerns like streaming, error handling, cost control, and structured output.

This is where "using AI" becomes "building with AI." By the end of this unit, you'll have two real tools — a network troubleshooter CLI and a security advisory summarizer — that call production APIs and handle edge cases that tutorials never mention.

**The "API wrapper" pattern:** Most AI applications are fundamentally API wrappers with good prompts and proper error handling. That's not an insult — it's a powerful pattern. Master it here.

---

## What You'll Build

- An **AI network troubleshooter CLI** with streaming responses and conversation memory
- A **security advisory summarizer** that fetches CVEs from NVD and generates daily digests
- A **multi-provider abstraction** — same interface for OpenAI, Anthropic, and Gemini
- **Cost tracking dashboard** — know what every tool call costs
